ThreeIconPlugin

The plugin is able to show three different icons.

Each icon can be set separately via the REST API.

Show icon in one of the three slots. Supported are bitmap (.bmp) and GIF (.gif) files.

The bitmap files are limited to:

  • 24 or 32 bits per pixel.
  • 1 plane.
  • No compression.

Note, if you are using _gimp_ to create bitmap files, please configure like:

  • Compatibility options: Don't write color informations.
  • Extended options: Select 24 bit per pixel.

REST API

Set icon

POST {{ORIGIN}}/rest/api/v1/display/uid/<PLUGIN-UID>/bitmap/<SLOT-ID>/<ICON-ID>
POST {{ORIGIN}}/rest/api/v1/display/alias/<PLUGIN-ALIAS>/bitmap/<SLOT-ID>/<ICON-ID>
  • PLUGIN-UID: The plugin unique id.
  • PLUGIN-ALIAS: The plugin alias name.
  • SLOT-ID: The id of the slot (starting with 0 from left ([0; 2])).
  • ICON-ID: The id of the icon ([0; 1]).

Clear icon

POST {{ORIGIN}}/rest/api/v1/display/uid/<PLUGIN-UID>/bitmap/<SLOT-ID>/<ICON-ID>?clear=<STATE>
POST {{ORIGIN}}/rest/api/v1/display/alias/<PLUGIN-ALIAS>/bitmap/<SLOT-ID>/<ICON-ID>?clear=<STATE>
  • PLUGIN-UID: The plugin unique id.
  • PLUGIN-ALIAS: The plugin alias name.
  • SLOT-ID: The id of the slot (starting with 0 from left ([0; 2])).
  • ICON-ID: The id of the icon ([0; 1]).
  • STATE: true or false

Control slot

GET {{ORIGIN}}/rest/api/v1/display/uid/<PLUGIN-UID>/slot/<SLOT-ID>
GET {{ORIGIN}}/rest/api/v1/display/alias/<PLUGIN-ALIAS>/slot/<SLOT-ID>
POST {{ORIGIN}}/rest/api/v1/display/uid/<PLUGIN-UID>/slot/<SLOT-ID>?activeIconId=<ACTIVE-ICON-ID>&icons._0_.iconid=<ICON-ID>&icons._0_.fullPath=<FULL-PATH>&icons._1_.iconid=<ICON-ID>&icons._1_.fullPath=<FULL-PATH>
POST {{ORIGIN}}/rest/api/v1/display/alias/<PLUGIN-ALIAS>/slot/<SLOT-ID>?activeIconId=<ACTIVE-ICON-ID>&icons._0_.iconid=<ICON-ID>&icons._0_.fullPath=<FULL-PATH>&icons._1_.iconid=<ICON-ID>&icons._1_.fullPath=<FULL-PATH>
  • PLUGIN-UID: The plugin unique id.
  • PLUGIN-ALIAS: The plugin alias name.
  • SLOT-ID: The id of the slot (starting with 0 from left ([0; 2])).
  • ACTIVE-ICON-ID: The id of the active icon in the slot ([0; 1])
  • ICON-ID: The id of the icon, the path relates to.
  • FULL-PATH: The full path to the icon image.
